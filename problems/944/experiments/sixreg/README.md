# Six-Regular Search for Erdős #944

This directory contains the finite checks for the Skottova-Steiner 2025
Problem 5.2 subproblem: whether a 6-regular `(4,1)` graph exists.

## Files

- `deg6_n11.cnf`, `deg6_n12.cnf`, `deg6_n13.cnf`, `deg6_n14.cnf`:
  CNF encodings for simple 6-regular graphs on `n` vertices.
- `sixreg_n11.out`, `sixreg_n12.out`, `sixreg_n13.out`:
  graph streams generated by `smsg.exe` from those CNFs.
- `check_stream2.cpp`:
  C++ stream checker. For each graph it classifies:
  - `threecol`: graph is 3-colourable;
  - `notVC`: graph is not 4-vertex-critical;
  - `vcWithCritEdge`: graph is 4-vertex-critical but has a critical edge;
  - `TARGET`: graph is a 6-regular `(4,1)` target.
- `check_n11_summary2.out`, `check_n12_summary2.out`,
  `check_n13_summary2.out`:
  final checker summaries.
- `unique_6reg_4vc_n13.txt`:
  the one `n=13` stream entry that is 6-regular and 4-vertex-critical; it has
  critical edges, so it is not a target.
- `n14_chunks`, `n14_chunks_v2`:
  two native chunked validations of the `n=14` slice. Aggregating the `.out`
  files in either directory gives
  `total=21609301 threecol=42667 notVC=21566634 vcWithCritEdge=0 TARGET=0 badline=0`.
- `enum_9shore.cpp`, `verify_9shore_survivor.py`,
  `kill_9shore_survivor.py`:
  computer-assisted exclusion of 9-vertex shores of nontrivial 6-edge-cuts in
  6-regular targets.
- `enum_10shore.cpp`, `verify_10shore.py`:
  computer-assisted exclusion of 10-vertex shores of nontrivial 6-edge-cuts in
  6-regular targets.
- `enum_shore.cpp`, `verify_shore_indep.py`, `shore12_chunks`,
  `shore13_chunks`, `shore14_chunks`:
  generalized computer-assisted exclusion of 11- through 14-vertex shores.

## Re-running the Checker

Compile:

```powershell
g++ -O3 -std=c++17 check_stream2.cpp -o check_stream2.exe
```

Check the existing streams:

```powershell
cmd /c "check_stream2.exe 11 < sixreg_n11.out > check_n11_summary2.out 2> check_n11_progress2.err"
cmd /c "check_stream2.exe 12 < sixreg_n12.out > check_n12_summary2.out 2> check_n12_progress2.err"
cmd /c "check_stream2.exe 13 < sixreg_n13.out > check_n13_summary2.out 2> check_n13_progress2.err"
```

Expected final summaries:

```text
n=11: total=266 threecol=3 notVC=263 vcWithCritEdge=0 TARGET=0
n=12: total=7849 threecol=50 notVC=7799 vcWithCritEdge=0 TARGET=0
n=13: total=367860 threecol=849 notVC=367010 vcWithCritEdge=1 TARGET=0
```

The checker reports progress to stderr every 10,000 input graphs.

## Re-generating the SMS Streams

The SMS executable used locally is:

```text
E:\Projects\ErdosProblems\sms-win\build\src\smsg.exe
```

Example command shape:

```powershell
E:\Projects\ErdosProblems\sms-win\build\src\smsg.exe --vertices 13 --dimacs deg6_n13.cnf --all-graphs > sixreg_n13.out 2> sixreg_n13.err
```

SMS may output isomorphic copies when its minimality cutoff is reached. This
does not affect the negative target check: every output graph is checked, and
duplicates cannot hide a target. For exact non-isomorphic counts, rerun SMS
with `--cutoff 0`.

## Verification Status

As of 2026-06-11, the verified finite conclusion is:

```text
No 6-regular (4,1) graph appears in the exhaustive n = 11, 12, 13, 14 checks.
```

This proves any 6-regular `(4,1)` graph, if one exists, has at least 15
vertices, assuming the CNF/SMS generation covers all simple 6-regular graphs
on the specified vertex set. The `n=14` conclusion is double-checked by two
different chunk partitions.

The verified local structural conclusion is:

```text
In a 6-regular (4,1) graph, every nontrivial 6-edge-cut shore has at least 15 vertices.
```

The 9..14-shore steps are machine-assisted and depend on 6-regularity; do not
state them for general minimum-degree-six targets without a separate argument.
