Scope and expected usefulness

The support-separation cut applies:

throughout this five-label core;

to every T-vertex, because c
0
	​

independently of H14 and independently of anti-tightness;

without any fixed A/B template or P-enumeration.

It is especially relevant to the leading profiles with ∣T∣=2 or 3. It converts the expensive chain

A/R codegree→A/B edge variables→state intersections

into a direct constraint on column-support overlaps.

For the cheapest experiment, add only the ordered T-pair tuple cut. For the stronger experiment, add the full isolated-T-to-all-R implications.

Weakest steps

The 6191 certificate uses the code convention ∣A∣=∣B∣=6, so that p=27 means exactly nine A/B nonedges. It should not be transferred unchanged to a convention where one side has seven actual state vertices.

common_a[t][r] and common_b[t][r] must equal the true support intersections. If they are merely lower bounds used inside another codegree expression, the proposed linearization is not safe.

The strengthening is a projection of constraints already logically present in the verifier. It can improve propagation substantially, but it does not shrink the mathematical feasible set beyond the current exact A/R and B/R codegree model.

The complete 6191 proof does not imply that the same four Φ-subsets close the other top profiles; its reusable part is the isolated-support-separation lemma.
